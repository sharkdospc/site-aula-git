Git e GitHub

Seu guia definitivo para controle de versão

O que é o Git?

O Git é um sistema de controle de versão distribuído, usado para rastrear mudanças no código fonte durante o desenvolvimento de software.

O que é o GitHub?

O GitHub é uma plataforma online que hospeda repositórios Git, permitindo colaboração, compartilhamento de código e controle de versões na nuvem.

Diferenças do Git e GitHub

  • Git: É um software de controle de versão instalado localmente.
  • GitHub: É um serviço web para hospedar repositórios Git e facilitar colaboração.

Principais comandos Linux para usar o Git Bash

  • git init – Inicia um repositório Git.
  • git clone [url] – Clona um repositório remoto.
  • git status – Verifica o status dos arquivos.
  • git add [arquivo] – Adiciona arquivos para o próximo commit.
  • git commit -m "mensagem" – Registra as mudanças.
  • git push – Envia alterações para o repositório remoto.
  • git pull – Puxa alterações do repositório remoto.

Como criar um repositório no GitHub?

  1. Acesse github.com e faça login.
  2. Clique no botão "New repository".
  3. Coloque um nome para o repositório e configure as opções.
  4. Clique em "Create repository".

Como ligar um projeto existente?

Entre na pasta do projeto no terminal e execute:

git init
git remote add origin [url-do-repositorio]

Como gerar uma versão com git?

Faça o commit das alterações usando:

git add .
git commit -m "Descrição da versão"

Como enviar para o GitHub?

Depois do commit, envie as alterações com:

git push origin master
